1.2.1 ISO C

The GNU C Library совместима со стандартом C, принятом American National Standards Institute (ANSI): American National Standard X3.159-1989—“ANSI C” и, позже, International Standardization Organization (ISO): ISO/IEC 9899:1990, “Programming languages—C”. Мы здесь ссылаемся на стандарт как ISO C т.к. это более главный стандарт касательно ратификации. Заголовочные файлы и средства библиотеки, которые состовляют the GNU C Library являются надмножеством того, что задается стандартом ISO C.

Если вы заинтересованы в строго мсоблюдении стандарта ISO C, вы должны использовать опцию ‘-ansi’ при компиляции вашей программы the GNU C compiler. Это говорит компилятору включать только возможности стандарта ISO в заголовочных файлах библиотеки, пока вы явноо не запросите дополнительные возможности. См. 1.3.4 Макросы тестов возможностей, о том, как это сделать.

Being able to restrict the library to include only ISO C features is important, потому что ISO C ограничивает имена, которые определяются реализацией библиотеки, и расширения GNU не подогнаны под эти ограничения. См. Зарезервированные имена, для большей информации об этих ограничениях.

Этот мануал не пытается дать вам все детали различий между ISO C и старыми диалектами. Он дает советы, как писать программы, работающие под множеством диалектов C, но полнота не является целью.
